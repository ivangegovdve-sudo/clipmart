## 2025-02-24 - O(N) Data Fetching Due To Missing Slug Column
**Learning:** The database schema for `creators` lacks a `slug` column, requiring the backend and server components to fetch all rows to calculate the slug dynamically with `slugify(entry.displayName)` and find the match. This is an O(N) performance and memory bottleneck, as the app scales fetching full rows containing all columns just to compute and filter by a string is highly unscalable.
**Action:** When working with dynamic route slugs that rely on display names without direct DB indexing, always optimize by fetching *only* the minimal necessary fields (e.g. `id` and `displayName`) to compute the slug matching in JavaScript, then perform a secondary indexed lookup by `id` to retrieve the full object. Alternatively, advocate for adding a dedicated `slug` column to the database schema.
