## 2024-03-12 - Memory limits when using find() with large tables
**Learning:** Using JS `find()` on `db.select().from(table)` pulls the entire table (all columns, all rows) into memory. This is especially dangerous when trying to match derived columns (like a slugified string) that cannot easily be indexed or queried natively in SQL without complex functions.
**Action:** When filtering by a derived/computed string and a pure SQL query is not feasible without schema changes, always perform a preliminary lightweight query fetching ONLY the `id` and the raw string needed to compute the match. Perform the JS `find()` on this lightweight dataset, then fetch the full record only for the matched `id`. This dramatically reduces network transfer size and application memory consumption.
